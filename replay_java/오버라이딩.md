# 오버라이딩

## 오버로딩과 오버라이딩의 차이점

```
- 오버로딩 : 같은이름의 메소드를 여러개 가지면서 매개변수(파라미터값,인자값,인수값)의 유형과 개수를 다르도록 하는 기술.
- 오버라이딩 : 상위(부모)클래스가 가지고 있는 메소드를 하위(자식)클래스가 재정의해서 사용.
```

```java
//부모클래스
class Woman{
    public String name;
    public int age;

    //info 메서드
    public void info(){
        System.out.println("여자의 이름은 "+name+", 나이는 "+age+"살입니다.");
    }

}
//자식클래스
class Job extends Woman{ //Woman클래스(부모클래스)를 상속받음 :

    String job;

    // 부모클래스(Woman)에 있는 메소드와 이름은 같지만 하위클래스(Job)에서 메소드를 재정의 한다.
    public void info() {//부모(Woman)클래스에 있는 info()메서드를 재정의
        super.info();
        System.out.println("여자의 직업은 "+job+"입니다.");
    }
}

public class OverTest {
//main클래스
    public static void main(String[] args) {

        //Job 객체 생성
        Job job = new Job();

        //변수 설정
        job.name = "유리";
        job.age = 30;
        job.job = "프로그래머";

        //호출
        job.info();

    }

}

// 실행결과 : "여자의 직업은 프로그래머(job)입니다."

// 오버라이딩을 안했을때 실행결과 : "여자의 이름은 유리(name), 나이는 30(age)살입니다."
```

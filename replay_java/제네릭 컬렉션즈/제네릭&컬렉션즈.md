## 컬렉션즈란? (=컬렉션즈 프레임워크)

```
* 컬렉션즈 프레임워크라는 것은 다른 말로는 컨테이너라고도 부른다.
    즉, 값을 담는 그릇이라는 의미.
    그런데 그 값의 성격에 따라서 컨테이너의 성격이 조금씩 달라진다.
    자바에서는 다양한 상황에서 사용할 수 있는 다양한 컨테이너를 제공하는데
    이것을 컬렉션즈 프레임워크라고 부른다.

Collection과 Map이라는 회상위 카테고리가 있고, 그 아래에 다양한 컬렉션들이 존재한다.

* Map            * Collection
-HashMap           - List(순서가 있는 저장공간) -> ArrayList
-Hashtable         - Set(집합적인 저장 공간) -> HashSet
-SortedMap         - Queue



***컬렉션 종류
    ArrayList, HashMap, HashSet이 대표적이고,
    이중에서 *ArrayList를 정말 많이 사용함.

```

## 기본배열과 컬렉션즈의 차이점

```
1. 배열은 크기가 고정되어있지만 arrayList는 사이즈가 동적인 배열이다.
2. 배열은 primitive type(int, byte, char 등)과 object 모두를 담을 수 있지만, arrayList는 object element만 담을 수 있다.
3. 배열은 제네릭을 사용할 수 없지만, arrayList는 타입 안정성을 보장해주는 제네릭을 사용할 수 있다.
4. 길이에 대해 배열은 length 변수를 쓰고, arrayList는 size() 메서드를 써야한다.
```

```
1. Array[]
길이확인 -> length사용
    1-1)장점
        - 인덱스를 통한 검색이 용이함.
        - 연속적이므로 메모리 관리가 편하다.
        - 데이터 조회가 빠름.
    1-2)단점
        - *크기가 고정되어 있기 때문에 어떤 엘리먼트가 삭제되면, 삭제된 상태를 빈 공간으로 남겨두어야 한다. => 메모리 낭비
        - 항상 초기 size를 설정해야만 한다.
        - 컴파일(배열 선언) 이후 배열의 크기를 변동 할 수 없다.(그러므로 크기를 확장해야할 경우 새로운 배영ㄹ을 생성하여 복사 해야한다.)
        - 크기를 처음부터 크게 만들면 메모리가 낭비 된다.

2. ArrayList<>
길이확인 -> size()사용
    2-1)장점
    - List(배열)의 크기를 임의로 변화 시킬 수 있다.(자동적으로 설정된다.)
    - 어떤 인덱스가 삭제되면, 삭제된 상태를 뒤에있던 인덱스들이 모두 앞으로 당겨서 빈공간을 채운다.
    - 그러므로 List에는 빈공간이 없어 메모리의 소모량을 줄일 수 있다.
    2-2)단점
    - 동적타입이라 크기가 유동적이다. 그래서 엘리먼트를 삭제하게 되면 기존의 데이터들의 순서가 바뀌게 된다.(조회할 때 번거로움)
    - List 비순차적으로 데이터를 추가, 삭제에 시간이 많이 걸린다.(단, *순차적으로 추가,삭제는 시간이 빠르다.)
```

# 팀명 : 조기취업

## 발표내용

취업시장은 현재 너무도 잘알고 있고 앞서 발표해주신분들이 현재 상황을 잘 설명 해주셨기 때문에 저희는 취업시장은 간단하게 조사하였고 개념과 장,단점을 위주로 조사를 하였습니다.

```

 -동적타입언어(변수의타입 지정X,node.js)
 ###장점###
    -변수의 선언과 사용이 쉽다.
    -쉽게 말하면 int에서 string형 변환시
    예를 들면 문자메소드를 이용한 타입변경을 안하여도 된다는 것.
    -변수이름을 선택할때 장애 발생 가능성이 줄어듬.
    -확장성이 좋음.
###단점###
    -인원이 많은 프로젝트,복잡한 코드작성 등 제약이 많음.
    -쿼리가 길어지다보면 애초에 신경을 쓴다고해도 변수의 타입이 무엇인지 몰라 코딩하는데 큰 어려움을 겪음
    ex)변수 a에 숫자 1을 넣어 놓고 .split()연산을 해도 경고 문구가 뜨지 않는다.

node.js
1. 보다 난이도가 낮음

2. 빠른 개발을 하는 것에 대해 상대적으로 유리함

3. 스타트업 기업들이 사용하는 추세 (파이썬과 더불어)

4. 자바보다는 좁은 취업문
```

```
 -정적타입언어(변수의타입 지정O,springboot)
###단점###
    -융통성이 없고 불편하다는 이야기가 많다.
    -까다롭다.
 ###장점###
    -java가 인기가 많고 쓰는 이유 중에 제일 큰 이유는 안정성이 검증 되었기 때문입니다.
    -동적언어와는 반대로 변수의 타입을 지정해주어야 하지만 변수의 타입을 알려주기 때문에 에러발생 가능성이 적습니다.
    코드를 깔끔하게 짤 수 있어서 처음 코드를 짤 때는 복잡하고 까다롭지만 유지보수를 하기에는 spring이 훨씬 효율적이라는 생각입니다.
    -에러 발생 가능성까지 다 조사하여 알려줌.

SPRING
1. 높은 퀄리티,안정성

2. 복잡한 시스템을 위한 복잡한 구조

3. 2번과 같은 이유로 난이도가 상대적으로 높음

3. 한국은 자바공화국 (취업시장 유리)
```

```
-결론은 백엔드를 중심으로 개발자가 되고 싶다면 spring을 먼저 배우자는게 저희팀의 생각입니다. 조사에 따르면 정적타입을 배운 후에 동적타입을 배우라는 이야기가 많았습니다.

이유1. 기본적으로 자료형의 개념에 대해 제대로 이해를 한 후에 동적타입을 배우는게 프로그램의 이해도가 높을 것이라는 생각입니다.

이유2. 정적타입이 고인물이라는 이야기가 많은데 나쁘게 말하면 고인물이지만 좋게 말하자면 정적타입을 알면 동적타입은 쉽게 이해하고 쉽게 접근 할 수 있는 이야기입니다.

TMI) 프론트엔드를 원하시는 분들은 node.js를 선택하는것이 더 유리할 것 같습니다. node.js의 태초는 프론트엔드를 중심으로 나온 것이다 보니 나중에 백엔드에도 관심을 가지게 된다면 node.js로 뒤쳐지지 않는 백엔드도 어느정도 커버를 할 수 있다고 생각합니다.
그리고 정말 좋은 개발자가 되려면 시대에 맞는 언어를 공부해야하고 시대에 뒤쳐지지 않게 시야를 넓게 봐야한다는게 시장조사를 하며 느낀점입니다.
```

- # 애플리케이션(소프트웨어)이 만들어 가는 과정

## 단어 해설

```
1. 애자일(= 애자일 방법론) :
    1-1) 고객의 변화하는 요구에 민첩하게 대응하는 운영 철학이나 문화(개발 프로그램X)
    1-2) 개발해 나가면서 필요한 것들을 피드백을 바탕으로 주도적으로 개발
2. 스크럼 :
    2-1) 스크럼은 애자일 방법론의 종류중 하나이다.
    2-2) 프로젝트 관리를 위한 상호 점진적 개발 방법론이다.
3. 스프린트 : 스크럼에서의 반복주기를 말한다.
4. 백로그 :
    4-1) 고객의 요구사항을 조사하여 구현해야 할 사항을 정의한 문서
    4-2) 개발해야 할 기능 또는 제품에서 요구하는 기능과 우선순위(기능명세서 작성)

```

```
* 애자일(Agile = 민첩한, 날렵한) 방법론(구글에서 창시)
(스타트업들이 애자일 방법론을 많이 함!)
    1. 애자일 방법론이란? 신속한 반복 작업을 통해 실제 작동 가능한 소프트웨어를 개발하여 지속적,유동적으로 제공하기 위한 소프트웨어 개발 방식
    많은 방법론 중에 **스크럼(Scrum)(=애자일 개발 프로세스)을 많이 사용!
    ex) 뱅크샐러드, toss, 애플 등...


애자일 방법론의 진행과정
    1. 요구분석(기능명세서, 권한 정책)
    2. 설계(ERD(==DB 테이블 명세서), UI 스토리보드, 클래스 명세서...)
    3. 구현 (퍼블리싱, API 개발(== 백엔드개발),  테이블 생성)
    4. 테스트 (optional) 개발기간에 따라 선택(필수X)
        4-1) 테스트를 하기위해 이용하는 도구 : Postman, JUnit, Jest ...
    5. 유지보수


스크럼은 애자일 방법론 과정의 하나.
아래와 같은 특징을 가짐!
(이런 과정들을 지라(ZIRA)에서 관리를 할 수 있음.
    1. 솔루션에 포함할 기능 / 개선점에 대한 우선 순위를 부여한다.
    2. ***개발주기(1~3번까지)가 30일 넘으면 안됨!
        2-1) 30일 넘으면 산으로 가고 있구나...
        2-2) 기능만 개발하면 됨! 완벽하지 않아도됨!
        2-3) 개발 주기(한달)마다 ***실제 동작할 수 있는 결과를 제공하라!
    3. 개발 주기(한달)마다 적용할 기능, 개선 목록을 제공하라!
    4. 날마다 15분 정도 회의를 가져라.
    5. 항상 팀 단위로 생각하라.
        5-1) 원할한 의사소통을 위하여, 구분 없는 열린 공간을 유지하라.
    6. ***기능 개발(소분류)은 3일 안쪽으로 개발! (3일 넘으면 산으로 감!)
    7. 개발할 제품에 대한 요구사항목록을 제품 백로그(Backlog)라고 한다.

결론 : 개발 주기(스프린트)가 모여 하나의 어플리케이션 탄생!
	*스프린트란? 짧은 거리(한달)를 전력 질주하다, 반복 주기를 의미함(평균적으로 1주~4주)
```

- ## 제품 백로그(Product Backlog) 예시

```
제품 백로그(Product Backlog) == 클라이언트의 요구사항분석
백로그를 작성하는 양식이 정해져 있는것은 아님.

ex)
프로젝트 명 : 학생전용 깃허브 플랫폼 제작(초,중,고학생전용)
(국가프로젝트라고 가정)

(백로그 작성 예시)
(백로그 작성할 때 주어가 필수!)
    1. 학생은 작성한 소스(코딩)를 업로드 해야한다.
    2. 학생은 로그인 후 본인 소스를 볼 수 있어야 한다.
    3. 학생은 업로드한 소스를 다운로드받을 수 있다.
    4. 선생님은 학생들 소스를 관리 및 확인할 수 있다.
    5. 선생님은 학생들 소스를 다운로드받을 수 있다.

1. 스프린트(04/12 ~ 05/06)
    목표 : 큰 기능을 우선 구현하자!

    기능명세서(엑셀로 작성 예정) 작성 후 우선 순위 결정!
        1. 소스 업로드 구현
        2. 소스 확인 구현
        3. 로그인 구현
        4. 소스 다운로드 구현
        5. 소스 관리 구현

```

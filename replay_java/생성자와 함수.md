# 데이터 타입

```
자바에서는 동적 데이터타입과 정적 데이터타입을 저장하는 공간이 다르다
.
1. 정적 타입(int, double, String, char, 배열...)

2. 동적 타입(String, class,  배열...)
    2-1) 동적 데이터타입 저장공간 = *Heap이라는 메모리 공간에 저장
    2-2) 동적타입을 관리하는 영역을 *Heap Area이라고 한다.
    2-3) 데이터가 저장될 때 고유번호를 받음.(그 고유번호가 메모리 주소.)
    2-4) 자바에서 String은 "=="로 비교 X, "=="은 고유번호를 받은 메모리 주소를 비교하는것.
    2-5) 자바에서 String은 .equals("비교할값")로 비교, (.equals("비교할값") == String의 내용 비교)
```

# 자바

```
- 계층 구조
    프로젝트 -> 패키지(default) -> 클래스파일

    1. 프로젝트 생성
        1-1) 프로젝트이름은 대문자 or 소문자
        1-2) 복합명사일 경우 ' _ ' 사용 (정답은 아님)

    2. 패키지 생성(생략가능)
        2-1) 패키지 이름은 소문자
        2-2) 복합명사일경우 카멜표기법 사용
    (프로젝트에 바로 클래스를 만들면 dafault패키지가 생성된다.)

    3. 클래스 파일 생성
        3-1) 클래스 이름은 항상 맨앞은 대문자 사용

- 현재 사용 자바 버전 = 12버전
   ( 하위버전을 사용하면 상위버전을 사용하지 못하지만 상위버전 사용시 하위버전 사용 가능)
```

## 생성자와 함수(메소드)의 차이

```
1. ***생성자
    1-1) return값이 없음(생성자는 매개 변수는 존재할 수 있지만 반환 값은 존재하지 않는다.)
    1-2) 클래스에 생성자를 명시하지 않아도 default로 생성자가 자동으로 생성되서 main메소드에서 호출 할 수 있다.
    1-3) 같은 클래스에 있는 생성자도 호출시 new를 사용하지만, main함수에 static함수를 이용해
        main함수와 같은 클래스에 있는 생성자에도 static을 명시하면 new를 사용하지 않고 호출할 수 있다.

2. 함수 : return 값이 있어야 사용 가능.(단, return값이 없는것 = 리턴값이 void일 경우)

3. 사용 목적
    3-1) 생성자 : 생성자는 값을 초기화하는게 목적
    3-2) 함수 : 함수는 파라미터에 값을 받아서 결과를 처리(return)하는게 목적

4. ***class파일에 생성자를 생성하지 않더라도 class를 main메소드에서 new로 호출할 수 있다.
    4-1) why? 별도의 생성자를 만들지 않았을 때 컴파일러에 의해 컴파일 타임에
              자동(default)으로 생성자가 만들어지기 때문이다. 이를 기본생성자라고 한다.
```

```java
        4-1 ex)
        public class DwAcademy{ // 클래스파일
        // public DwAcademy(){ } 기본생성자가 생략되어 있는것.
            int x;
            String y;

            public static void main(String[] args){
                DwAcademy academy = new DwAcademy()
                academy.x = 0;
                academy.y = "홍길동";
            }
        }
```

## new 호출

```
- new를 이용해서 클래스를 호출할 때 생성자가 자동으로 호출됨.
- new로 불러온 변수 재활용 가능!
```

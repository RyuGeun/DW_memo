- # 추상화 개념

```
(추상화는 인터페이스를 사용하기 위해 알아야함.)
1. 공통적으로 사용하는 기능을 따로 정의.

```

- ## \*\* "추상"이라는 단어는 공통 기능(공통으로 사용되는 클래스, 메소드라고 알면 됨.)

- ## 추상클래스(미완성 설계도)

```
1. 추상클래스는 "abstract"라는 키워드로 선언된 클래스.
    1-1) 추상클래스에는 일반메소드, 추상메소드를 정의할 수 있다.

2. 추상클래스는 new로 호출(인스턴스 화)할 수 없다.
    2-1) 추상클래스는 상속을 목적으로 둔다.
```

- ## 추상메소드

```
(메소드를 정의 할경우에도 abstract를 선언하여 메소드 정의)
1. 추상메소드는 구현이 목적이 아님.(공통 기능인 메소드의 title만 정의한다.(로직 구현X))
    1-1) 추상메소드는 슈퍼클래스 or 추상클래스에서 선언부만 작성한 후 서브클래스에서 오버라이딩하여 로직을 구현한다.
2. ***추상메소드가 있는 추상클래스를 상속받으면 무조건 서브클래스에는 모든 추상메소드를 무조건 오버라이딩을 해야함.
```

- ## 추상클래스의 한계

```
1.  클래스이기에 다중상속 X
2. 인터페이스는 다중상속 가능O
```

```java
package 추상화;

abstract class Chef{// 추상 클래스
	//**추상 메소드 or 일반메소드를 정의할 수 있음(abstract키워드로 선언된 클래스 안에)
	public void 요리를하다() {//일반 메소드

	}

	//추상메소드와 일반메소드의 차이점 : 중괄호가 없음.
	//추상메소드의 목적 : 서브클래스에서 오버라이딩이 목적임.
	abstract public void 비법을전수하다();//추상메소드
}

class Food extends Chef{
	//Food가 오버라이딩을하면 Food클래스는 인스턴스화(== new 호출) 할 수 있다.
	@Override
	public void 비법을전수하다() {
		if(2<4) {
			//로직구현
		}

	}

}

public class 추상화를배우자 {
	public static void main(String[] args) {
		Food f = new Food();
	}
}

```
